# Critical Path - Work Backward from a Traction Goal

> **Up:** [[Home|Home]] → [[04-Frameworks-and-Mental-Models/Frameworks and Mental Models Index|Frameworks and Mental Models]] → [[04-Frameworks-and-Mental-Models/Focus-Execution-and-Systems/Focus Execution and Systems Map|Focus, Execution, and Systems]]

## One-line summary
Define the traction result that would materially change the company, work backward to the few necessary milestones, and reject work that does not advance the next constraint.

## Context
Startups face more plausible projects than they can execute, and “important” work easily displaces the work that determines survival. A traction goal and its critical path convert a vague growth ambition into a short, revisable sequence of necessary outcomes.

## Key insights
- **A traction goal must move the needle.** It should be quantitative, time-bound, and significant enough to change the company’s options—not a vanity target that can be reached without altering its trajectory.
- **Milestones are outcomes, not task piles.** Working backward exposes the few conditions that must become true before the goal can be achieved.
- **The next milestone governs focus.** Only work required to reach that milestone belongs on the immediate path; attractive partnerships, features, and campaigns can still be strategically wrong now.
- **The path is a hypothesis.** New evidence may change the route, but revision should follow measured learning rather than boredom, fear, or a stakeholder’s newest idea.
- **Constraints make saying no easier.** A shared path turns prioritization from a debate about preferences into a test of causal necessity.

## Framework / model (if applicable)
1. **Set the destination:** choose a measurable traction goal tied to the company’s current phase and survival or strategic options.
2. **Work backward:** list the minimum intermediate outcomes required immediately before the goal, then repeat until reaching the current state.
3. **Name the next constraint:** identify the one milestone presently blocking progress.
4. **Filter the work:** accept only actions that directly improve that constraint or supply essential learning about it.
5. **Review after evidence:** when a milestone is reached or disproven, redraw the remaining path from the new reality.

## Tactics / how to apply
- Write the goal as “Reach X by date Y so that Z becomes possible,” making both magnitude and strategic significance explicit.
- Ask of every proposed project: “Which critical-path milestone does this advance, and how will we know?”
- Keep a visible not-now list so rejected ideas are preserved without stealing current capacity.
- Break distant targets into traction subgoals small enough to govern weekly work, while retaining the same causal chain.
- At each review, separate failure of execution from failure of the path hypothesis before changing direction.

## Notable examples
DuckDuckGo set a daily-search target that would make the company meaningfully fundable, then worked backward through the product and distribution milestones needed to reach it. Half.com used the same logic before launch: the site had to function, sufficient inventory had to exist, and distribution partnerships had to be in place; unrelated deals were distractions even when prestigious.
