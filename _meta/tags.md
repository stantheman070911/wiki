# Tags

The tag vocabulary for THE WIKI. Every tag used on a knowledge page must appear in this file — `npm run check` enforces it. Tags carry a facet prefix: `topic/` (what it is about), `person/` (the source author), `source/` (a program, podcast, or brand used as provenance).

To add a tag: reuse an existing term if one fits; otherwise add a line to the right section in this file in the same edit that uses it. Avoid near-synonyms of existing tags.

## Boundaries

Disambiguation notes for overlapping terms:

- `topic/simplicity` is the quality or principle of being simple; `topic/simplification` is the act or process of reducing complexity.
- `topic/avatar` is a designed customer/content archetype; `topic/persona` is the public identity or character presented by a creator or brand.
- `topic/owned-audience` is any directly reachable audience asset; `topic/private-domain` is the Chinese private-traffic operating model and is narrower.
- `topic/short-form-video` is the cross-platform format; platform tags such as `topic/tiktok` are used only when platform mechanics matter.
- `topic/leadership` concerns direction and influence; `topic/management` concerns planning and control; `topic/org-design` concerns roles, structure, decision rights, and coordination.
- Use `person/` and `source/` tags only when that entity is part of the page provenance, not for people or brands mentioned merely as examples.
