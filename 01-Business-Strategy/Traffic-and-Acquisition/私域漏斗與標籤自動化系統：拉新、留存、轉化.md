# 私域漏斗與標籤自動化系統：拉新、留存、轉化

> **Up:** [[Home|Home]] → [[01-Business-Strategy/Business Strategy Index|Business Strategy]]

## 一句話總結

用 EDM／Landing Page 拉新，靠下 tag 把名單分眾，再用「日常手動＋系列自動化」的內容矩陣持續留存與轉化——這是 [[私域經營三段邏輯：引流、留存、轉換]] 的具體落地版本。

## 背景

本條目將「私域漏斗與標籤自動化系統：拉新、留存、轉化」整理為可重複使用的strategy，以下保留原有框架、步驟與例證。

## 核心策略

### 漏斗總覽：拉新 → 留存 → 轉化

```
EDM → Landing Page → 名單磁鐵 → 下tag
下tag → 日常手動化（進行式）→ 日常更新文
下tag → 系列自動化（完成式）→ 新手包自動化 ／ 系列文自動化 ／ 促銷型自動化
```

| 階段 | 內容 |
|---|---|
| 拉新 | EDM → Landing Page |
| 留存 | 名單磁鐵 → 下 tag |
| 留存、轉化 | 系列自動化 |

**EDM 指標：** 開信率（AVG OPEN RATE）、點擊率（AVG CLICK RATE）、抵達率。

### 流量分佈

```
EDM → Landing Page → 各流量渠道
```
Landing Page 的導流去向：公域-社群流量（Instagram／Facebook／Podcast）、公域-個人網站（品牌網站／銷售網站）、社群流量-私域（LINE 社群）、廣告流量。

### 標籤與自動化系統

**下 tag 的四種分類邏輯：**
- 常見 QA
- 不同管道（IG／FB／活動：免費活動／付費活動）
- 不同 TA（已購買／未購買——未購買中再分「參加免費活動→潛在用戶」與「未參加免費活動→沈睡用戶」）
- 不同需求（創作／創業／生活）

**系統自動化：**
- 日常手動化（進行式）→ 日常更新文。
- 系列自動化（完成式）：
  - 新手包自動化 → 仙人掌型內容／玫瑰花型內容 → **關於我**（我在做什麼／我是誰／我提供的服務）
  - 系列文自動化 → 仙人掌型內容 → **關於產品**（常見 QA／產品介紹）
  - 促銷型自動化 → 含羞草型內容（教育型內容＋銷售型內容，搭配檔期）
  - 另有「受歡迎文章」形成社群 top 10 榜單

**內容型別對應意圖：** 仙人掌型＝定位／專業（關於我、關於產品）；玫瑰花型＝形象／人設；含羞草型＝促銷（教育＋銷售，搭配檔期）。這是一套與陳修平「七大內容主題」互補的分類視角——七大主題按內容題材分類，這裡按「在漏斗中的功能」分類，兩者可以交叉標記同一支內容。

### 需要留意的框架張力：線性漏斗 vs. 非線性信任

本文與 [[私域經營三段邏輯：引流、留存、轉換]] 都把顧客旅程畫成一條線性管線（拉新→留存→轉化），並用自動化序列去執行。但 [[品牌即媒體公司：注意力到信任框架]] 明確主張「行銷漏斗已死／非線性」，信任是透過眾多探索觸點累積而成，不是一條可預測的輸送帶。

這兩種說法不必然互斥：**「漏斗已死」談的是你腦中對顧客心理歷程的假設**——不要以為陌生人看一次廣告就會照著階段乖乖往下走；**本文與私域經營三段邏輯談的是執行層的自動化系統**——一旦有人主動選擇加入名單（opt-in），你仍然需要一套機制去持續、分眾地跟進。實務上，可以同時抱持「信任是非線性累積的」信念，同時在背後跑著線性的 email／標籤自動化——但如果你把這套流程圖當成「顧客一定會照這樣走」的心智模型，就該用品牌即媒體公司那篇的觀點提醒自己：多數人不會乖乖照著箭頭走，自動化只是輔助，不是保證。

## 應用

- 幫既有的名單設計至少四種 tag 維度（管道／TA／需求／購買狀態），才能對不同名單發送真正對應的自動化內容，而不是對所有人發同一封信。
- 把「新手包」「系列文」「促銷」三種自動化內容分開規劃，各自對應「關於我」「關於產品」「促銷」三種不同目的，避免促銷內容過早出現在還不認識你的新名單面前。
- 定期用 EDM 的開信率／點擊率檢查每個自動化序列的健康度，而不是設定後就永久不再檢視。
